Three workflows
Webhook · Haiku

Lead Qualification

HMAC-signed lead in → Claude scores fit, intent, urgency → Slack alert for qualified leads → logged to Postgres. Forged signature → clean 401.

Cron · Opus

Daily Content Brief

RSS at 09:00, deduped by GUID → Opus critic selects and summarizes the few items that matter → posted to Slack, archived to Postgres.

IMAP · Haiku

Email Classification

Poll inbox → Claude classifies into sales / support / recruiting / billing / other → routes each to its own Slack channel → logged.

Shipped, not prototyped
✓Full observability — every LLM call writes input, output, tokens, duration, and cost to Supabase.
✓Validated outputs — Zod-equivalent schema checks; malformed responses fail loudly, not silently.
✓Bounded retry — max two attempts on every external call; no runaway cost loops.
✓Signed webhooks — HMAC-SHA256, constant-time compare, clean 401 before any model call.
✓No silent failures — every error writes error_log and pages a Slack channel.
✓Clean git history — single author, conventional commits, hook-enforced.
